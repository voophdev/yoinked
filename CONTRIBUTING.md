# Contributing

Thanks for wanting to help! Small, focused contributions are the sweetest.

How to contribute

1. Fork the repo and create a feature branch: `git checkout -b feat/your-change`
2. Write tests for any new behavior and run the test suite: `pytest -q`
3. Keep commits small and focused; use a clear PR description
4. Open a pull request against `main` and reference any related issues

Coding style

- Keep code simple and readable.
- Follow existing project patterns.
- No unnecessary refactors in the same PR as the feature.

Communication

- Add a short description of what you changed and why in the PR body.
- If your change affects configuration or behaviour, include examples.

Thanks — maintainers will review and give feedback. ❤️
